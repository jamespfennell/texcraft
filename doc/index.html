Texcraft

Texcraft is a project to create a new, modern implementation of TeX. The name is a portmanteau of TeX and Oxide, the latter referencing the programming language used in the project, Rust. The project is currently in a very early prototyping/proof-of-concept phase.

Project description

Texcraft is not a specific distribution of TeX. Instead, it is directly inspired by LLVM and is an attempt to implement TeX as a collection of loosely coupled libraries. A library in Texcraft might be:

  • A new pagination algorithm
  • Handling for a particular font format.
  • Implementations of a subset of TeX primitives, like the various \def commands.

These components can then be combined in a modular way to create a specific TeX distribution, just like pieces of LLVM can be combined to create a specific compiler. Parts of Texcraft could also be used to create other software exposing some limited TeX functionality - for example, a web service that validates TeX formulas using the Texcraft formula parser.

This is a new approach to implementing TeX. The existing distributions all have separate monolithic codebases that are hard to reuse. There are many benefits to an LLVM-style approach:

  • It will be significantly easier to create new TeX distributions or to extend existing ones with new features; for example, grid based typesetting. While Texcraft sounds like a pure software project, one of the main motivations is facilitating innovation on the typesetting side.
  • Distributions can share code and features easily.
  • Texcraft components can be used in other contexts too, not just in TeX compilers.

While a modular architecture is key to these goals, we also think that Texcraft being written in a modern programming language and style will help significantly, too.

The main goal of Texcraft is make TeX implementation development easier, but there is also an opportunity here to fix common UX issues:

  • Documents will be automatically recompiled if the internal references change, and this will extend to BibTeX.
  • Errors will have clearer messages, will not trap users in confusing interactive prompts, and in many cases will auto-suggest a solution:
